Question-1: What are the different ways to select an element in the DOM?

Different ways to select an element in the DOM:

  • getElementById("id") – Selects an element by its unique ID.
  • getElementsByClassName("class") – Selects elements with a specific class and returns an HTMLCollection.
  • getElementsByTagName("tag") – Selects elements by their tag name and returns an HTMLCollection.
  • querySelector("selector") – Selects the first element that matches a CSS selector.
  • querySelectorAll("selector") – Selects all elements that match a CSS selector and returns a NodeList.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

Property Description
innerHTML Returns or sets the HTML content, including HTML tags.
innerText Returns or sets the visible text, excluding hidden text and HTML tags.
textContent Returns or sets the text content, including hidden text, but excluding HTML tags.
Performance `innerText` is slower than `textContent` because it accounts for CSS styles.

Question-3: What is event delegation in the DOM?

Event delegation is a technique in the DOM where a single event listener is attached to a parent element instead of individual child elements. This allows the parent to handle events for dynamically added or existing child elements. It relies on the event's bubbling phase to propagate up from the target element to the parent. This method improves performance, especially when there are many child elements. It is commonly used for handling events like clicks on lists or buttons.

Question-4: What is event bubbling in the DOM?

Event bubbling is a concept in the DOM (Document Object Model) where an event starts at the most specific element (the target) and then "bubbles" up to the root of the document, triggering any event listeners on ancestor elements. It is the default propagation behavior for most events. Developers can stop this propagation using the stopPropagation method. Event bubbling allows parent elements to handle events for child elements without needing direct listeners on each child. It contrasts with event capturing, where the event starts from the root and moves down to the target.
